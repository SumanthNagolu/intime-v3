These are **absolutely required** for the app to work:

### 1. NEXT_PUBLIC_SUPABASE_URL
- **Value:** Your Supabase project URL
- **Example:** `https://xyzabc123.supabase.co`
- **Get from:** https://app.supabase.com/project/_/settings/api
- **Add to:** ‚úÖ Production, ‚úÖ Preview, ‚ö†Ô∏è Development (optional)

### 2. NEXT_PUBLIC_SUPABASE_ANON_KEY
- **Value:** Your Supabase anonymous/public key
- **Example:** `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`
- **Get from:** https://app.supabase.com/project/_/settings/api (look for "anon public")
- **Add to:** ‚úÖ Production, ‚úÖ Preview, ‚ö†Ô∏è Development (optional)

### 3. SUPABASE_SERVICE_ROLE_KEY
- **Value:** Your Supabase service role key (‚ö†Ô∏è **SECRET** - never expose to client)
- **Example:** `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`
- **Get from:** https://app.supabase.com/project/_/settings/api (look for "service_role")
- **Add to:** ‚úÖ Production, ‚úÖ Preview, ‚ö†Ô∏è Development (optional)
- **‚ö†Ô∏è IMPORTANT:** This is a sensitive key with full database access!

- **Value:** Your production app URL

## ‚úÖ Quick Add Instructions

### Method 1: Automated Script (Recommended)

```bash
bash scripts/add-vercel-env-vars.sh
```

This interactive script will prompt you for each value.

### Method 2: Manual via Dashboard

1. **Go to:** https://vercel.com/team_OTUu5KkOBTEFrGO5RjFdSlSY/intime-v3/settings/environment-variables

2. **Click:** "Add New" ‚Üí "Environment Variable"

3. **For each variable:**
   - **Key:** Variable name (e.g., `NEXT_PUBLIC_SUPABASE_URL`)
   - **Value:** Actual value
   - **Environments:** Select:
     - ‚úÖ Production
     - ‚úÖ Preview
     - ‚ö†Ô∏è Development (optional)

4. **Click:** "Save"

### Method 3: Vercel CLI

```bash
# Add to production
echo "your_value" | vercel env add VARIABLE_NAME production

# Add to preview
echo "your_value" | vercel env add VARIABLE_NAME preview

# Add to all environments
echo "your_value" | vercel env add VARIABLE_NAME production preview development
```
