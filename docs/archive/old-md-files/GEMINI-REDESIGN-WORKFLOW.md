
### 4.2 Component-by-Component Migration

**Example: Navbar**

**Before (Current Earthy Theme):**
```tsx
<nav className="fixed top-0 bg-charcoal text-ivory shadow-lg">
  <button className="px-6 py-2 bg-rust hover:bg-rust/90 rounded-full">
    Login
  </button>
</nav>
```

**After (New Theme from Gemini):**
```tsx
<nav className="fixed top-0 bg-primary-700 text-white shadow-xl">
  <button className="px-6 py-2 bg-accent-500 hover:bg-accent-600 rounded-lg">
    Login
  </button>
</nav>
```

**Migration Steps:**
1. Open component file
2. Find and replace color classes:
   - `bg-charcoal` → `bg-primary-700` (or Gemini's equivalent)
   - `bg-ivory` → `bg-white` or `bg-gray-50`
   - `bg-rust` → `bg-accent-500`
   - `text-stone-600` → `text-gray-600`
   - `rounded-full` → `rounded-lg` (if Gemini uses less rounded)
3. Update font weights if changed
4. Test in browser

### 4.3 Automated Find & Replace

**Create this script:** `/frontend-prototype/migrate-theme.sh`

```bash
#!/bin/bash

# Replace old colors with new colors across all components
# CUSTOMIZE THESE BASED ON GEMINI'S OUTPUT

find components -type f -name "*.tsx" -exec sed -i '' \
  -e 's/bg-charcoal/bg-primary-900/g' \
  -e 's/bg-ivory/bg-gray-50/g' \
  -e 's/bg-rust/bg-accent-500/g' \
  -e 's/text-charcoal/text-gray-900/g' \
  -e 's/text-stone-600/text-gray-600/g' \
  -e 's/border-stone-200/border-gray-200/g' \
  -e 's/rounded-full/rounded-lg/g' \
  {} +

echo "Theme migration complete! Review changes and test."
```

**Run:**
```bash
chmod +x migrate-theme.sh
./migrate-theme.sh
```

## Best Practices for Gemini Design Workflow

### Do's ✅
1. **Be specific** - Describe exact use case and context
2. **Provide examples** - Show Gemini your current component for reference
3. **Iterate in rounds** - Design system → Components → Pages
4. **Ask for code** - Always request React + Tailwind code
5. **Request mobile-responsive** - Specify Tailwind breakpoints
6. **Export design tokens** - Get colors, spacing as variables

### Don'ts ❌
1. Don't ask for everything at once - Too broad, lower quality
2. Don't skip the design system phase - Consistency matters
3. Don't forget accessibility - Ask Gemini to include ARIA labels
4. Don't ignore mobile - Always specify responsive requirements
5. Don't skip iteration - First draft is rarely perfect
