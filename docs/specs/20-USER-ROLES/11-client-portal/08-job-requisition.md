## Overview

This use case describes the complete job requisition workflow from the client perspective, enabling hiring managers to create, submit, and track staffing requests through the Client Portal. The workflow includes automatic approval routing, budget validation, requisition templates, and integration with InTime's recruiting pipeline.

**Key Objectives:**
- Self-service job requisition creation
- Automated approval workflows
- Budget and compliance validation
- Real-time status tracking
- Template library for common roles
- Seamless handoff to InTime recruiting team

---

## Actors

**Primary:**
- Hiring Manager (creates and manages requisitions)
- Client Admin (configures workflows and templates)

**Secondary:**
- Department Head (approves requisitions)
- Procurement Manager (approves high-value requisitions)
- HR Contact (compliance review)
- InTime Account Manager (receives approved requisitions)
- InTime Recruiter (begins fulfillment)

**System:**
- Approval routing engine
- Budget validation service
- Email notification service
- Audit logging service

---

## Preconditions

1. User authenticated as Hiring Manager or Client Admin
2. User has job requisition creation permission
3. Department and cost center configured
4. Approval workflows configured
5. Budget codes available (if required)

---

## Trigger

**Primary Triggers:**
- Hiring Manager clicks "Create Job Requisition"
- Hiring Manager uses requisition template
- System prompts for renewal of expiring contract position

## Alternative Flows

### ALT-1: Use Template

**Flow:**
1. Select "Use Template" on start screen
2. Choose from template library (pre-configured roles)
3. Template pre-fills job info, requirements, skills
4. User reviews and customizes as needed
5. Updates budget and start date
6. Proceeds through workflow as normal
7. Time saved: ~50% faster than from scratch

### ALT-2: Copy Existing Requisition

**Flow:**
1. Select "Copy from Existing Job"
2. Search for previous requisition
3. System duplicates all fields except:
   - Start date (reset to future)
   - Budget year (update to current)
   - Status (reset to draft)
4. User updates as needed
5. Submit for new approval cycle

### ALT-3: Bulk Requisition Creation

**Flow:**
1. Client Admin uses bulk upload
2. Download CSV template
3. Fill multiple requisitions in spreadsheet
4. Upload CSV file
5. System validates all rows
6. Preview requisitions before creation
7. Confirm and create all at once
8. Each enters approval workflow independently

## Business Rules

### BR-1: Requisition Lifecycle

**States:**
- Draft: Being created, can edit freely
- Pending Approval: In approval workflow, read-only
- Approved: All approvals complete, handed to InTime
- Active: InTime recruiting in progress
- On Hold: Temporarily paused
- Filled: All positions filled
- Cancelled: Withdrawn or rejected
- Expired: Not filled within timeout (90 days default)

### BR-2: Approval Requirements

**Automatic Routing Based on:**
- Department (determines department head)
- Total estimated cost (triggers procurement)
- Contract duration (> 12 months triggers executive review)
- Bill rate (> $200/hr triggers executive review)
- Rush priority (adds COO approval)

**Minimum Required:**
- All requisitions: Department head approval
- > $150K total cost: Procurement approval
- > $300K total cost: CFO approval
- New department/cost center: HR approval

### BR-3: Budget Validation

- Must have valid cost center
- Estimated cost cannot exceed available budget (unless override approved)
- Budget checked in real-time
- Q1-Q4 budget periods respected
- Fiscal year boundaries enforced
- Budget reservations when approved (funds held)

### BR-4: Edit Restrictions

**Can Edit:**
- Draft status: Full edit capability
- Pending Approval: Can withdraw and edit
- Approved: Can request changes (requires re-approval)

**Cannot Edit:**
- Active with candidate submissions
- Filled or Closed status
- Historical data (after 90 days)

### BR-5: Template Requirements

- Templates managed by Client Admin only
- Must include all required fields
- Can have department-specific templates
- Version controlled (track changes)
- Can be shared across organization

### BR-6: Market Rate Guidance

- System provides market rate data based on:
  - Job title and skills
  - Location (if on-site/hybrid)
  - Experience level required
  - Current market conditions
- Data refreshed monthly
- Sources: Industry surveys, InTime placement data
- Guidance only, not binding
