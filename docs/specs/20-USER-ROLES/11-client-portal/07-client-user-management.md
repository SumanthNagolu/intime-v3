## Overview

This use case describes the complete user management workflow for Client Admin users, enabling them to manage their organization's portal access. Admins can invite new users, modify roles and permissions, suspend or deactivate accounts, configure approval workflows, and manage team structure - all without requiring InTime staff intervention.

**Key Objectives:**
- Self-service user lifecycle management (invite, update, deactivate)
- Role-based access control configuration
- Team and department structure management
- Approval workflow customization
- Audit trail and compliance tracking

---

## Actors

**Primary:**
- Client Admin (user with administrative privileges)

**Secondary:**
- Hiring Manager (manages own team members)
- Client Users (being managed)
- InTime Account Manager (support escalation)

**System:**
- Email notification service
- Audit logging service
- SSO provider (if configured)
- Permission management service

---

## Preconditions

1. User is authenticated as Client Admin
2. Organization has active InTime account
3. License capacity available for new users
4. User management permissions enabled
5. Email delivery system operational

---

## Trigger

**Primary Triggers:**
- Client Admin clicks "Manage Users" from portal navigation
- Client Admin needs to invite new team member
- Client Admin needs to modify user role or permissions
- Client Admin needs to deactivate departed employee

## Alternative Flows

### ALT-1: SSO Auto-Provisioning

**Flow:**
1. User logs in via SSO for first time
2. Identity provider returns SAML assertion with attributes
3. InTime auto-creates user account based on attributes
4. Role assigned based on group membership or attribute mapping
5. Department assigned based on directory attribute
6. User lands on quick profile completion page
7. User can immediately access portal

**Attribute Mapping Example:**
```

  "Hiring-Managers"         â†’ hiring_manager
  "Portal-Users"            â†’ client_user
```

### ALT-2: Self-Service Role Upgrade Request

**Flow:**
1. Client User clicks "Request Additional Access"
2. User selects desired role and provides justification
3. System routes request to Client Admin for approval
4. Client Admin reviews request and user activity
5. Client Admin approves/denies with optional message
6. User notified of decision
7. If approved, role updated and user re-authenticates

### ALT-3: Emergency Access Revocation

**Flow:**
1. Client Admin needs to immediately revoke user access
2. Client Admin clicks "Emergency Suspend" from user list
3. Minimal confirmation modal (reason optional)
4. All user sessions terminated immediately
5. User locked out within 30 seconds
6. Admin can provide details and reassignment later
7. InTime Account Manager notified of emergency action

2. Invitations sent to new users via email
3. Role changes applied and user permissions updated
4. Audit log entries created for all changes
5. Notifications sent to affected users
6. License seat count updated
7. Approval workflows routing correctly
8. SSO attribute mappings synchronized

## Business Rules

### BR-1: User Limits

- Minimum 1 Client Admin per organization
- Maximum users based on license tier
- Inactive users (90+ days) eligible for auto-suspension (opt-in)
- Suspended users count toward license limit
- Deactivated users do not count toward limit

### BR-2: Role Hierarchy

**Permission Levels (ascending):**
1. Client User (read-only, feedback)
2. Hiring Manager (create jobs, schedule interviews, approve placements)
3. Procurement Manager (approve high-value requisitions, view billing)
4. HR Contact (compliance, documentation, approvals)
5. Client Admin (full access, user management, configuration)

**Rules:**
- Higher roles include all lower role permissions
- Role changes require re-authentication
- Only Client Admin can promote to Client Admin
- Cannot demote yourself to non-admin if last admin

### BR-3: Email Requirements

- Must be valid email format
- Corporate domain preferred (warning if external)
- Must be unique within organization
- Cannot use disposable email services
- Email verification required before access

### BR-4: Suspension vs Deactivation

**Suspension:**
- Temporary access revocation
- Preserves all data and settings
- Can be reactivated by any admin
- Optional auto-reactivation date
- Counts toward license limit

**Deactivation:**
- Permanent removal (requires support to reverse)
- Preserves historical data (compliance)
- Releases license seat immediately
- Cannot log in (shows "account deactivated")
- Requires confirmation with typed username

### BR-5: Approval Workflows

- Each workflow must have at least one approver
- Maximum 5 approval levels per workflow
- Timeout must be 1-30 business days
- Auto-escalation optional but recommended
- Conditional triggers supported (amount, department, etc.)

### BR-6: Bulk Operations

- CSV import maximum 500 users per batch
- Validation required before processing
- Partial success allowed (skip invalid rows)
- Summary report generated after import
- Failed rows exported for correction

### BR-7: Audit Requirements

**Logged Events:**
- User invited (by whom, when, role)
- User activated (first login)
- Role changed (from â†’ to, reason)
- User suspended (reason, by whom)
- User deactivated (reason, last day)
- Permission changed
- Workflow modified

**Retention:** 7 years for compliance

## Integration Points

### Email Service
- **Provider:** SendGrid / AWS SES
- **Templates:**
  - User invitation
  - Role change notification
  - Suspension notification
  - Deactivation confirmation
  - Bulk import summary
- **Tracking:** Delivery status, opens, clicks

### Identity Provider (SSO)
- **Auto-Provisioning:** Create users on first SSO login
- **Attribute Sync:** Update user info from directory
- **Group Mapping:** Map AD/LDAP groups to roles
- **Deprovisioning:** Detect removed users and suspend

### Audit Service
- **Events:** All user lifecycle changes
- **Retention:** 7 years
- **Export:** CSV, JSON formats
- **Alerting:** Anomaly detection (mass deletions, etc.)

### License Management
- **Real-time Tracking:** Current seat usage
- **Alerts:** 90% capacity warning
- **Overage:** Allow temporary overage (billing adjustment)
- **Sync:** Update billing system on changes

## Security

### Access Control

**Client Admin:**
- Full user management
- Configure workflows
- View all audit logs
- Manage integrations

**Hiring Manager:**
- View users in own department
- Cannot modify users
- Can request role upgrades for team

**Client User:**
- View own profile
- Update contact info
- Cannot view other users

### Data Protection

**Sensitive Data:**
- Passwords never shown or exported
- MFA secrets encrypted
- Suspension reasons visible to admins only
- Audit logs access-controlled

**Session Security:**
- Role changes require re-authentication
- Session timeout after 30 minutes idle
- Active sessions revokable
- Concurrent session limit (5)

### Audit Trail

**Logged Events:**
- User invited (by, when, role, email)
- User activated (first login, IP, device)
- Role changed (from, to, reason, by)
- Permissions modified (what, by, when)
- User suspended (reason, by, reassignments)
- User deactivated (reason, by, data handling)
- Workflow configured (changes, by)
- Bulk import (file, results, by)

**Compliance:**
- SOC 2 Type II
- GDPR compliant (right to erasure)
- HIPAA ready (audit logging)
- Retention: 7 years
