# Use Case: Regional Director Dashboard - Monitor Regional Performance

## Overview

| Property | Value |
|----------|-------|
| Use Case ID | UC-RD-002 |
| Actor | Regional Director |
| Goal | Monitor real-time regional performance across all countries, pods, and key metrics |
| Frequency | Multiple times daily (5-10 views per day) |
| Estimated Time | 5-15 minutes per view |
| Priority | Critical |

---

## Preconditions

1. User is logged in as Regional Director
2. User has `regional_dashboard.view` permission
3. Regional scope is configured (AMERICAS, EMEA, or APAC)
4. Data synchronization is current (last sync < 15 minutes)
5. At least one country is active in the region

---

## Trigger

One of the following:
- Start of workday (daily routine)
- Before important meetings (CFO review, board presentation)
- Alert notification received (critical metric threshold exceeded)
- Mid-day performance check
- End-of-day review
- Ad-hoc strategic analysis

---

## Main Flow

### Step 1: Navigate to Regional Dashboard

**User Action:** Click "Regional Dashboard" in navigation or use Command+D shortcut

**System Response:**
- URL changes to: `/employee/workspace/regional-dashboard`
- Loading state shows skeleton (500ms)
- Dashboard widgets populate from top to bottom
- Real-time data streams in via WebSocket

## Alternative Flows

### A1: Drill Down to Country Dashboard

**Trigger:** User clicks "View Details" on a specific country in the Country Performance section

**Flow:**
1. System navigates to country-specific dashboard
2. URL changes to: `/employee/workspace/country-dashboard/usa`
3. Country-level scorecard displays
4. Country manager information shown
5. Pod breakdown for that country only
6. Country-specific alerts and issues

### A2: Export Dashboard to PDF/PowerPoint

**Trigger:** User clicks "Export" or "Share Report"

**Flow:**
1. Export modal opens with options:
   - Format: PDF, PowerPoint, Excel
   - Sections: Select which widgets to include
   - Date Range: MTD, QTD, YTD, Custom
   - Recipients: Email distribution list
2. User selects options and clicks "Generate"
3. System generates report (10-30 seconds)
4. Download link appears or email sent
5. Report includes:
   - Executive summary
   - All selected dashboard widgets
   - Charts and graphs
   - Data tables
   - Timestamp and generated-by info

### A3: Set Up Custom Alerts

**Trigger:** User clicks "Configure Alerts" in dashboard settings

**Flow:**
1. Alert configuration modal opens
2. User can set threshold-based alerts:
   - Revenue < Target by X%
   - Margin below Y%
   - Utilization outside range
   - Pod performance degradation
   - Critical deals at risk
3. Notification preferences:
   - Email, Slack, SMS, In-app
   - Frequency: Real-time, Daily digest, Weekly summary
4. Save alert rules
5. System monitors and sends notifications when triggered

### A4: Compare Time Periods

**Trigger:** User selects "Compare To" option in Quick Filters

**Flow:**
1. Comparison period selector appears
2. User selects: Prior Month, Prior Year, Budget, Custom Date Range
3. Dashboard widgets update to show side-by-side comparison
4. Variance calculations displayed
5. Charts show overlaid trend lines
6. Color coding for positive/negative variances
