## Part 1: Role Overview

### Your Mission

**Ensure every placed consultant succeeds, and every client is delighted.**

You are the guardian of our placements. Once a consultant starts:
- You ensure smooth onboarding
- You track their performance
- You handle any issues
- You secure renewals and extensions

**Retention is revenue:** Every month a consultant stays placed = continuous billing.

### Key Responsibilities

| Area | What You Own |
|------|--------------|
| **Onboarding** | Ensure smooth day-1 experience |
| **Check-ins** | Regular pulse with consultants and clients |
| **Issue Resolution** | Handle problems before they escalate |
| **Performance** | Monitor and improve consultant delivery |
| **Renewals** | Secure contract extensions |
| **Off-boarding** | Manage graceful exits |

### Success Metrics

| Metric | Target | Frequency |
|--------|--------|-----------|
| Consultant check-ins | 100% coverage weekly | Weekly |
| Client satisfaction | 4.5+/5 rating | Monthly |
| Issue resolution time | <24 hours | Per issue |
| Renewal rate | 85%+ | Quarterly |
| Consultant retention | 90%+ complete contract | Quarterly |
| Extension rate | 60%+ | Quarterly |

### Compensation Structure

| Component | Amount |
|-----------|--------|
| Base salary | ₹1,00,000-1,50,000/month |
| Retention bonus | ₹5,000 per consultant/month retained |
| Extension bonus | ₹10,000 per extension secured |
| Target total | ₹1,50,000-2,50,000/month |

## Part 2: Understanding Delivery

### What is Delivery Management?

**Delivery = Everything after placement, before off-boarding**

Your job is to maximize the value of each placement by:
1. Ensuring consultant performs well
2. Keeping client happy
3. Extending contracts when possible
4. Managing smooth transitions

### The Delivery Lifecycle

```
Placement → Day 1 → Week 1 → Month 1 → Ongoing → Renewal → Extension/Exit
     │         │        │         │         │         │           │
     └─────────┴────────┴─────────┴─────────┴─────────┴───────────┘
                    DELIVERY MANAGER OWNS THIS JOURNEY
```

### Why Delivery Matters

**For Consultants:**
- Clear expectations
- Support when struggling
- Career development
- Reference for future

**For Clients:**
- Quality assurance
- Single point of contact
- Issue resolution
- Partnership feeling

**For InTime:**
- Retention = recurring revenue
- Referrals from happy clients
- Lower acquisition costs
- Reputation building

## Part 4: Consultant Onboarding

### Before Day 1

**One Week Before:**
- [ ] Confirm start date with all parties
- [ ] Get client onboarding info (building, parking, contacts)
- [ ] Prepare welcome kit (if applicable)
- [ ] Schedule Day 1 check-in call
- [ ] Set up in tracking system

**Day Before:**
- [ ] Reminder call to consultant
- [ ] Confirm logistics (time, location, contact)
- [ ] Share any last-minute details
- [ ] Wish them luck!

### Day 1 Check-In

**Call Agenda (15 min):**
```
1. "How did it go? Did you find everything okay?"
2. "Did you meet your manager? First impressions?"
3. "Any issues with access, systems, logistics?"
4. "What are you working on first?"
5. "Any questions or concerns?"
6. "I'll check in again [day]. Call me if you need anything."
```

### First Week Follow-Up

**Day 3 Check-In:**
- Settling in okay?
- Getting work assigned?
- Any access issues?
- Team dynamics good?

**Day 5 Check-In:**
- Week 1 summary
- Challenges faced?
- Feedback from manager (if any)?
- Anything I can help with?

### First Month Milestones

| Week | Check-In Focus |
|------|----------------|
| Week 1 | Logistics, access, first impressions |
| Week 2 | Work assignments, team integration |
| Week 3 | Performance feedback, challenges |
| Week 4 | 30-day review, any concerns |

### Check-In Questions

**Standard Check-In:**

## Part 6: Issue Resolution

### Issue Categories

| Category | Examples | Urgency |
|----------|----------|---------|
| **Critical** | Termination risk, major conflict | Immediate |
| **High** | Performance concerns, dissatisfaction | Same day |
| **Medium** | Minor issues, questions | 24-48 hours |
| **Low** | Feedback, suggestions | Next check-in |

### Issue Resolution Process

**Step 1: Gather Information**
- Talk to consultant
- Talk to client (if appropriate)
- Document both sides
- Identify root cause

**Step 2: Assess & Plan**
- Determine severity
- Identify options
- Plan resolution approach
- Get approval if needed

**Step 3: Execute**
- Communicate plan to all parties
- Take action
- Document everything
- Set follow-up timeline

**Step 4: Follow Up**
- Confirm resolution
- Check for recurrence
- Update records
- Share learnings

### Common Issues & Resolutions

**Issue: Consultant Underperforming**
```
1. Get specific feedback from client
2. Share feedback with consultant (coaching approach)
3. Create improvement plan with milestones
4. Weekly progress check-ins
5. Escalate if no improvement in 2 weeks
```

**Issue: Consultant Unhappy**
```
1. Listen fully, understand root cause
2. Validate their feelings
3. Discuss options (address issue, reassignment, etc.)
4. Involve client if appropriate
5. Monitor closely
```

**Issue: Client Unhappy**
```
1. Listen and apologize for their experience
2. Document specific concerns
3. Address with consultant immediately
4. Propose solution (coaching, support, replacement)
5. Follow up frequently until resolved
```

**Issue: Consultant Wants to Leave**
```
1. Understand why (better offer, unhappy, personal)
2. See if issue can be resolved
3. If not, plan transition
4. Notify client professionally
5. Find replacement quickly
```

### Escalation Matrix

| Situation | Escalate To | Timeframe |
|-----------|-------------|-----------|
| Performance concerns | Recruiting Lead | 24 hours |
| Client complaint | Account Manager + Founder | Immediate |
| Consultant wants out | Recruiting Lead | Same day |
| Contract at risk | Founder | Immediate |

### Performance Conversations

**Positive Feedback:**
```
"[Name], I wanted to share that [Client] gave you great
feedback this month. Specifically, they mentioned [specific
praise]. Keep up the great work!

Is there anything you need from me to continue succeeding?"
```

**Constructive Feedback:**
```
"[Name], I have some feedback to share. [Client] mentioned
they'd like to see improvement in [area].

Specifically, [concrete example].

What's your perspective on this? What support do you need
to address it?

Let's check in again next week to see how things are going."
```

### Performance Improvement Plan (PIP)

**When to Use:**
- Consistent negative feedback
- Multiple issues raised
- Risk of termination

**PIP Structure:**
1. Clear statement of issues
2. Specific expectations
3. Support provided
4. Timeline (usually 2-4 weeks)
5. Consequences if not met

## Part 8: Renewals & Extensions

### Renewal Timeline

| Timing | Action |
|--------|--------|
| -60 days | Internal review (should we pursue?) |
| -45 days | Client temperature check |
| -30 days | Formal renewal discussion |
| -21 days | Rate negotiation (if needed) |
| -14 days | Contract extension sent |
| -7 days | Final confirmation |
| Day 0 | New contract starts |

### Renewal Conversation

**Client Call:**
```
"[Name], [Consultant]'s contract is coming up for renewal
in [X weeks]. Before we discuss next steps, I wanted to
check in:

1. How has [Consultant] been performing overall?
2. Do you have continued need for this role?
3. Any feedback for the next engagement period?

Based on your answers, would you like to extend the contract?"
```

### Handling Rate Discussions

**If Client Asks for Discount:**
```
"I understand budget is always a consideration. Let me
share our perspective:

[Consultant] has been delivering strong results -
[specific examples]. Their market rate has actually
increased since we started.

I can offer [small concession] as a loyalty gesture.
Would that work for you?"
```

**If Consultant Asks for Raise:**
```
"I appreciate you bringing this up. Let me look into
what might be possible.

Before I go to the client, help me understand:
- What value have you added that justifies an increase?
- What rate are you looking for?

Let me see what I can do and get back to you by [date]."
```

### Extension Success Factors

| Factor | How to Influence |
|--------|------------------|
| Performance | Regular coaching, quick issue resolution |
| Relationship | Stay visible, add value beyond basics |
| Timing | Start conversations early, don't wait |
| Value | Document and communicate consultant wins |

## Part 9: Off-boarding

### Planned Exit (Contract End)

**Two Weeks Before:**
- Confirm last day with all parties
- Ensure knowledge transfer happening
- Collect any company assets
- Schedule exit conversations

**Exit Conversation with Consultant:**
```
"As your contract wraps up, I wanted to thank you for your
work with [Client]. A few items:

1. How was your overall experience?
2. Any feedback on how we could improve?
3. Would you like us to find your next opportunity?
4. Can we use [Client] as a reference?

I'll send you an exit survey as well. Thank you!"
```

**Exit Conversation with Client:**
```
"As [Consultant]'s engagement ends, I wanted to thank you
for the partnership.

1. Overall, how was your experience with [Consultant]?
2. Any feedback for us?
3. Do you have other roles where we might help?
4. Would you be open to being a reference?

I'll stay in touch for future opportunities."
```

### Unplanned Exit (Termination/Resignation)

**If Consultant is Terminated:**
1. Get full details from client
2. Support consultant professionally
3. Begin replacement search immediately
4. Learn from what went wrong

**If Consultant Resigns:**
1. Understand reasons
2. Negotiate notice period if possible
3. Communicate professionally to client
4. Begin replacement search
5. Ensure smooth transition

### Off-boarding Checklist

```
□ Last day confirmed
□ Knowledge transfer complete
□ Assets returned
□ Final timesheet submitted
□ Exit survey sent
□ Reference request made
□ Thank you note sent
□ System records updated
□ Replacement (if needed) identified
```

## Part 11: Common Scenarios

### Scenario: Consultant No-Show

**Situation:** Consultant doesn't show up on Day 1.

**Response:**
1. Call consultant immediately
2. Find out reason
3. Notify client with explanation
4. Reschedule or find replacement
5. Document incident

### Scenario: Client Wants Immediate Termination

**Situation:** Client calls demanding consultant be removed today.

**Response:**
```
"I understand you're frustrated. Let me address this immediately.

Can you share what happened? [Listen]

I want to make this right. Here are our options:
1. I speak with [Consultant] and address the issue
2. We transition them out and find a replacement
3. [Other option based on situation]

What would you prefer?"
```

### Scenario: Consultant Competing Offer

**Situation:** Consultant received higher offer elsewhere.

**Response:**
```
"Thanks for being transparent with me about this.

Before you decide, let me see what we can do. Can you share:
- What rate are they offering?
- When do they need an answer?

Let me speak with the client and get back to you by [time].
In the meantime, please don't accept anything yet."
```
