## Operating Model Overview

### Three Talent Sources
```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   ACADEMY   │    │  RECRUITING │    │  CAMPAIGNS  │
│ Train new   │    │ Source exp. │    │ Inbound     │
│ talent      │    │ talent      │    │ attraction  │
└──────┬──────┘    └──────┬──────┘    └──────┬──────┘
       │                  │                  │
       └──────────────────┴──────────────────┘
                          │
                    ┌─────▼─────┐
                    │  TALENT   │
                    │   POOL    │
                    └─────┬─────┘
                          │
       ┌──────────────────┼──────────────────┐
       │                  │                  │
┌──────▼──────┐    ┌──────▼──────┐    ┌──────▼──────┐
│    BENCH    │    │  CONTRACT   │    │   DIRECT    │
│    SALES    │    │ PLACEMENT   │    │    HIRE     │
└─────────────┘    └─────────────┘    └─────────────┘
```

### Seven Funnels
| # | Funnel | Owner | Purpose |
|---|--------|-------|---------|
| 1 | Academy | Training Team | Train new IT professionals |
| 2 | Recruiting | Recruiters | Source experienced talent |
| 3 | Campaigns | Marketing | Attract inbound candidates |
| 4 | Account Acquisition | BDM | Win new clients |
| 5 | Job Fulfillment | Recruiters | Fill client positions |
| 6 | Bench Sales | Bench Team | Place available consultants |
| 7 | Delivery | Delivery Mgr | Ensure engagement success |

## Document Standards

### File Naming
- ALL_CAPS_WITH_UNDERSCORES.md for documents
- lowercase-with-dashes/ for folders
- Prefix with numbers for ordering (01-, 02-)

### Document Structure
Every handbook follows:
1. Role Overview (mission, responsibilities)
2. Success Metrics (how you're measured)
3. Day in the Life (what you actually do)
4. Key Processes (step-by-step SOPs)
5. Common Scenarios (how to handle situations)
6. Career Path (growth opportunities)

### Version Control
- All documents live in git
- Changes require PR review
- Major updates noted in document header
- Last updated date in document

---

## Contributing

### Adding New Documents
1. Follow the folder structure
2. Use existing documents as templates
3. Link from relevant index files
4. Update this README if adding sections

### Updating Existing Documents
1. Keep changes focused
2. Update "Last Updated" date
3. Note breaking changes in PR
