# Consultant Onboarding Checklist

**Document Type:** Operational Template
**Audience:** Delivery Managers, Operations Coordinators, Recruiters
**Last Updated:** February 2026
**Version:** 1.0

---

## Overview

This checklist ensures every placed consultant has a smooth transition from offer acceptance to productive client engagement. It is the responsibility of the **Delivery Manager** to oversee the entire process, coordinating across Recruiting, Operations, and the Consultant.

**Key Success Metrics:**
- Zero missed first-day logistics
- 100% compliance documentation completed pre-start
- Same-day check-in confirmation
- 0 unresolved onboarding issues by Day 2

---

## PRE-START PHASE (Days -7 to -1)

### Offer Letter & Contract Execution
- [ ] **Recruiter:** Send offer letter with all terms clearly outlined
  - Bill rate (to client)
  - Consultant rate (payment structure - W2, 1099, C2C)
  - Contract duration and renewal terms
  - Start date and location
  - Reporting manager/contact
- [ ] **Consultant:** Review, sign, and return executed offer letter
- [ ] **Recruiting:** Maintain signed copy in candidate file
- [ ] **Delivery Manager:** Receive countersigned copy; file in engagement folder
- [ ] **Status Update:** Notify Operations and Client of confirmed engagement

---

### Background Check & Drug Screening Coordination
- [ ] **Operations Coordinator:** Initiate third-party background check service
  - Standard background check + drug screening required
  - E-Verify clearance for all placements
- [ ] **Consultant:** Complete background check questionnaire and drug screening kit
  - Provide necessary identification and history information
  - Schedule/attend drug screening appointment
- [ ] **Operations:** Receive cleared/pending notification from screening provider
  - Flag any red flags or pending items
  - Do NOT allow consultant to start until cleared (unless client approves conditional employment)
- [ ] **Delivery Manager:** Confirm background clear status with client before Day 1

---

### Client-Specific Compliance Requirements
- [ ] **Delivery Manager:** Request compliance requirements from client
  - Security clearance requirements (if government/defense)
  - Additional screenings (credit, financial, industry-specific)
  - Training certifications or prerequisites
  - NDA/confidentiality tiers
  - Code of conduct acknowledgment
- [ ] **Consultant:** Complete all client-mandated compliance requirements
- [ ] **Operations:** Track and document all compliance proof of completion
- [ ] **Delivery Manager:** Confirm 100% compliance with client before start date

---

### Equipment & Access Requests
- [ ] **Delivery Manager:** Obtain from client:
  - Hardware configuration (laptop model, monitor, peripherals)
  - Software licenses needed (development tools, IDEs, databases)
  - VPN and network access requirements
  - Phone/mobile device policy
  - Parking and badge access details
- [ ] **Operations Coordinator:** Coordinate with client IT:
  - Hardware pre-staging at client location (or shipping to consultant home)
  - User account creation in all systems
  - Access provisioning (AD, applications, databases, repositories)
  - VPN credentials setup
  - Confirmation of ready-state (TEST: login verification required)
- [ ] **Client IT:** Confirm all systems are accessible and functional
- [ ] **Delivery Manager:** Document equipment baseline (serial numbers, software versions)

---

### First Day Logistics
- [ ] **Delivery Manager:** Confirm with client:
  - [ ] Client site address and directions
  - [ ] Parking instructions (location, cost, permit process)
  - [ ] Building access (badge, lobby procedures, security gates)
  - [ ] Check-in procedure (front desk, security, team location)
  - [ ] Primary contact name and phone number
  - [ ] First-day meeting/orientation agenda
  - [ ] Dress code (casual, business casual, formal, on-site specific)
  - [ ] Meal/break room access
  - [ ] Hours of operation and flex time policy
- [ ] **Delivery Manager:** Brief consultant on all logistics
  - Provide written one-page logistics summary
  - Send calendar invite for check-in call time
  - Provide client contact info and his/her direct phone
- [ ] **Consultant:** Acknowledge receipt of logistics; confirm arrival plan

---

### Timesheet System Setup & Training
- [ ] **Operations Coordinator:** Provision consultant in InTime platform
  - Create consultant profile with employment type (W2, 1099, C2C)
  - Configure project/engagement code
  - Set approval workflow (manager, delivery manager)
  - Test login credentials
- [ ] **Consultant:** Receive login credentials and platform documentation
  - Complete platform orientation walkthrough (30 min video/call)
  - Submit test timesheet in staging environment
  - Confirm understanding of weekly submission deadline
- [ ] **Operations:** Verify test submission received and accurate
- [ ] **Delivery Manager:** Send timesheet reminder calendar invite (weekly, same day/time)

---

### Benefits Enrollment (If Applicable)
- [ ] **Determine Eligibility:** W2 employee = benefits eligible; 1099/C2C = not eligible
- [ ] **Operations Coordinator (W2 Only):**
  - [ ] Send benefits package summary
  - [ ] Health insurance options with enrollment deadline
  - [ ] 401(k) plan info and enrollment window
  - [ ] PTO policy (accrual, carryover, blackout dates)
  - [ ] Other benefits (FSA, HSA, life insurance, disability)
- [ ] **Consultant (W2 Only):** Complete enrollment within deadline (typically 30 days)
- [ ] **Payroll:** Confirm benefits elections in HRIS system
- [ ] **Delivery Manager:** Maintain benefits confirmation in file

---

### Emergency Contact Collection
- [ ] **Operations Coordinator:** Send emergency contact form
  - Primary contact name, phone, relationship
  - Secondary contact name, phone, relationship
  - Medical conditions or allergies to disclose
- [ ] **Consultant:** Return completed form
- [ ] **Operations:** File form in secure location; provide copy to client safety/HR

---

### NDA & IP Agreement Execution
- [ ] **Delivery Manager:** Obtain required agreements from client
  - Client NDA template (if required)
  - InTime standard NDA (if client has none)
  - IP assignment agreement (ownership of work product)
  - Non-competition clause (if applicable)
- [ ] **Consultant:** Review and execute all agreements
  - Flag any concerns to Delivery Manager before signing
  - Return fully executed copies
- [ ] **Delivery Manager:** Send executed copies to client
  - File in consultant engagement folder
  - Calendar reminder to audit compliance (90 days post-start)

---

### Project Briefing with Delivery Manager
- [ ] **Delivery Manager:** Schedule pre-start briefing call (30-60 min)
  - Review project scope and objectives
  - Discuss team structure and reporting line
  - Clarify key technologies and tooling
  - Address any training gaps identified
  - Explain InTime's ongoing support/escalation process
  - Confirm consultant readiness for Day 1
- [ ] **Delivery Manager:** Send written briefing summary
  - Project overview (1 paragraph)
  - Key contacts and org chart
  - Technical stack summary
  - Success criteria for first 30 days
- [ ] **Consultant:** Complete any recommended pre-work or reading

---

## DAY 1 (Start Date)

### Morning Check-In Call
- [ ] **Delivery Manager:** Call consultant within 1 hour of their start time
  - [ ] Confirm safe arrival at client site
  - [ ] Confirm client check-in completed
  - [ ] Verify hardware and access are functional
  - [ ] Confirm first meeting/orientation scheduled
  - [ ] Assess initial impressions and any immediate concerns
  - [ ] Provide direct phone number for emergency escalation
  - [ ] Set time for end-of-day check-in call
- [ ] **Consultant:** Confirm ready status and ask clarifying questions

### Client Introduction Confirmation
- [ ] **Client Manager:** Greet consultant upon arrival
  - [ ] Confirm identity and background check status
  - [ ] Provide building orientation and safety briefing
  - [ ] Introduce to immediate team
  - [ ] Demonstrate workspace and equipment
  - [ ] Provide contact list and org chart
  - [ ] Confirm project assignment and first tasks
- [ ] **Client Manager:** Notify Delivery Manager that consultant successfully integrated

### System Access Verification
- [ ] **Client IT or Consultant:** Test access to:
  - [ ] Email account (send/receive test)
  - [ ] VPN (if remote or partial remote work)
  - [ ] Development environment or project repositories
  - [ ] Time tracking platform (InTime)
  - [ ] Collaboration tools (Slack, Teams, Jira, etc.)
  - [ ] Required databases or data sources
- [ ] **Consultant:** Report any access failures immediately to Delivery Manager
- [ ] **Delivery Manager:** Escalate access issues to Client IT; track resolution

### First Timesheet Reminder
- [ ] **Operations Coordinator:** Send automated reminder email with:
  - [ ] Link to InTime timesheet portal
  - [ ] Weekly submission deadline and approval workflow
  - [ ] Sample timesheet with hours breakdown
  - [ ] Escalation contact if technical issues arise
- [ ] **Consultant:** Bookmark timesheet portal for easy access

### Issue Escalation (If Problems Arise)
- [ ] **Consultant:** Report any critical issue (no access, no equipment, no meeting, safety concern) to Delivery Manager immediately
- [ ] **Delivery Manager:**
  - [ ] Respond to escalation within 30 minutes
  - [ ] Coordinate resolution with client
  - [ ] Provide workaround if needed (e.g., temporary equipment, remote work)
  - [ ] Document issue and resolution in engagement log
- [ ] **Delivery Manager:** Confirm resolution before end of business day

### End-of-Day Check-In Call
- [ ] **Delivery Manager:** Call consultant before COB
  - [ ] Ask about overall Day 1 experience
  - [ ] Confirm no open issues or blockers
  - [ ] Assess confidence level heading into Day 2
  - [ ] Provide encouragement and reiterate support availability
  - [ ] Schedule Week 1 daily check-in times
- [ ] **Consultant:** Provide initial feedback and ask follow-up questions

---

## WEEK 1 (Days 2-5)

### Daily Check-In Calls
- [ ] **Delivery Manager:** Call consultant same time each day (5-10 min calls)
  - [ ] Confirm all systems working and no blockers
  - [ ] Assess consultant productivity and integration
  - [ ] Gather initial feedback on team dynamics
  - [ ] Build rapport and provide ongoing encouragement
  - [ ] Flag any early red flags (performance, culture fit, technical gaps)
- [ ] **Consultant:** Provide honest daily feedback; escalate any concerns immediately

---

### 3-Day Feedback from Client (Day 3)
- [ ] **Delivery Manager:** Call client manager for early feedback
  - [ ] Is consultant showing up on time?
  - [ ] Technical capabilities matching expectations?
  - [ ] Demonstrated willingness to learn and take direction?
  - [ ] Any immediate concerns or issues?
  - [ ] Any early wins or positive contributions?
- [ ] **Delivery Manager:** Summarize feedback and share positive items with consultant
  - [ ] Address any concerns with consultant in supportive way
  - [ ] Provide guidance on any gaps identified

---

### 5-Day Formal Check-In with Client Manager (Day 5)
- [ ] **Delivery Manager:** Schedule formal 30-min call with client manager
  - [ ] Structured feedback on consultant performance across:
    - Technical competency
    - Communication and collaboration
    - Work quality and attention to detail
    - Initiative and problem-solving
    - Culture and team fit
  - [ ] Any changes needed to role or expectations?
  - [ ] Confirm continuation to Day 30
  - [ ] Early extension or end-of-contract discussion (if applicable)
- [ ] **Delivery Manager:** Document feedback in engagement file
- [ ] **Delivery Manager:** Share constructive feedback with consultant
  - [ ] Celebrate wins from Week 1
  - [ ] Provide coaching on any gaps
  - [ ] Establish expectations for Week 2-4

---

### Timesheet Submission Verification
- [ ] **Operations Coordinator:** Confirm timesheet submitted by deadline each day
  - [ ] Verify hours accuracy (should match work schedule)
  - [ ] Flag missing timesheets to consultant immediately
  - [ ] Send reminder email if not submitted by EOD Friday
- [ ] **Delivery Manager:** Review and approve timesheet before invoicing
  - [ ] Verify hours are reasonable and documented
  - [ ] Flag any unusual patterns (excessive OT, gaps, etc.)
- [ ] **Client Manager:** Approve timesheet to validate hours worked

---

### Issues or Concerns Log
- [ ] **Delivery Manager:** Create and maintain running log of any issues:
  - [ ] Technical blockers (access, equipment, dependencies)
  - [ ] Performance gaps or concerns
  - [ ] Client feedback (positive and negative)
  - [ ] Consultant feedback about engagement
  - [ ] Decisions made and follow-up actions
- [ ] **Delivery Manager:** Review log weekly with Operations and Recruiting
  - [ ] Escalate critical issues to leadership
  - [ ] Track resolution status

---

## 30-DAY REVIEW (Day 30 +/- 2 days)

### Performance Feedback from Client
- [ ] **Delivery Manager:** Schedule 45-min call with client manager
  - [ ] Comprehensive review across all competencies
  - [ ] Is consultant meeting role requirements?
  - [ ] Performance trajectory (improving, stable, declining)?
  - [ ] Quality of work product (code, deliverables, documentation)?
  - [ ] Collaboration and communication effectiveness?
  - [ ] Cultural fit and team integration?
  - [ ] Recommendation: Continue, improve performance, or end engagement?
- [ ] **Delivery Manager:** Document detailed feedback in performance file

---

### Consultant Satisfaction Survey
- [ ] **Operations Coordinator:** Send anonymous feedback survey to consultant
  - [ ] Are you receiving adequate support from InTime?
  - [ ] Is the client environment productive and respectful?
  - [ ] Are there any barriers to your success?
  - [ ] Would you recommend InTime to other consultants?
  - [ ] Any suggestions for improvement?
  - [ ] Preferred contact frequency going forward (weekly, bi-weekly, monthly)?
- [ ] **Consultant:** Complete and return survey (same week)
- [ ] **Delivery Manager:** Review feedback and address any concerns raised

---

### Rate/Contract Review
- [ ] **Delivery Manager:** Confirm with client:
  - [ ] Consultant compensation still aligned with market?
  - [ ] Any rate changes needed (positive or negative)?
  - [ ] Contract renewal terms (same rates, escalation, renewal duration)?
- [ ] **Recruiter:** Validate rates against current market rates
  - [ ] Confirm InTime margin is appropriate (35-45% typical)
  - [ ] Identify any rate adjustment needs
- [ ] **Delivery Manager:** Update contract terms in InTime system
  - [ ] Document any rate changes or extensions
  - [ ] Set renewal/extension review calendar date (90 days before contract end)

---

### Relationship Health Assessment
- [ ] **Delivery Manager:** Assess overall 30-day relationship health:
  - **Green (Healthy):** No issues, strong performance, client satisfied, consultant engaged
  - **Yellow (Monitor):** Minor issues or performance gaps, being addressed
  - **Red (At Risk):** Significant issues, performance below expectations, or cultural misalignment
- [ ] **Delivery Manager:** Create written 30-day summary:
  - Overall status (Green/Yellow/Red)
  - Key accomplishments and wins
  - Areas for improvement
  - Plan for next 30 days (if continuing)
  - Risk mitigation if Yellow/Red status
- [ ] **Delivery Manager:** Share summary with consultant (constructive, supportive tone)

---

## ONGOING (Monthly Cadence)

### Weekly Check-In Schedule
- [ ] **Delivery Manager:** Establish standing weekly call with consultant (30 min)
  - [ ] Assess productivity and any blockers
  - [ ] Discuss project progress and upcoming work
  - [ ] Address any concerns or feedback
  - [ ] Provide coaching and development guidance
- [ ] **Consultant:** Come prepared with updates and questions
- [ ] **Delivery Manager:** Calendar recurring meeting; build relationship

---

### Monthly Performance Reviews
- [ ] **Delivery Manager:** Schedule monthly 30-min call with client manager
  - [ ] Consultant performance review against role expectations
  - [ ] Delivery of commitments and project milestones
  - [ ] Quality and professionalism assessment
  - [ ] Client satisfaction rating (1-10 scale)
  - [ ] Feedback for consultant development
  - [ ] Early identification of any renewal risks
- [ ] **Delivery Manager:** Document performance and file notes

---

### Timesheet Compliance Tracking
- [ ] **Operations Coordinator:** Monitor weekly timesheet submission rates
  - [ ] Flag non-compliance immediately (missing timesheets, late submissions)
  - [ ] Send reminder emails for missed deadlines
  - [ ] Track repeat offenders
- [ ] **Delivery Manager:** Review timesheet accuracy and patterns
  - [ ] Flag unusual patterns (excessive OT, gaps, inconsistent hours)
  - [ ] Discuss with consultant if concerns arise
- [ ] **Billing:** Ensure timesheets are invoiced on schedule
  - [ ] Track client payment (monitor for slow-pay issues)

---

### Issue Resolution Log
- [ ] **Delivery Manager:** Maintain ongoing log of all issues
  - [ ] Technical blockers
  - [ ] Performance concerns
  - [ ] Client feedback (positive and negative)
  - [ ] Consultant requests or concerns
  - [ ] Actions taken and resolution status
- [ ] **Leadership:** Review log monthly to identify patterns
  - [ ] If multiple issues → escalate to account leadership
  - [ ] If consultant struggling → provide additional support/coaching
  - [ ] If client difficult → assess account viability

---

### Renewal/Extension Tracking (90 Days Before End)
- [ ] **Delivery Manager:** Identify contracts ending within 90 days
- [ ] **Recruiter:** Flag for potential renewal outreach
- [ ] **Delivery Manager:** Proactively contact client 90 days before end date
  - [ ] Confirm renewal intent (continue, extend, modify rate)?
  - [ ] Confirm continued need for consultant role?
  - [ ] Identify any known end date (project completion, budget, etc.)?
- [ ] **Delivery Manager:** Notify consultant of renewal status:
  - [ ] If renewing → confirm continuation terms and provide peace of mind
  - [ ] If ending → provide 30+ days notice to allow career planning
- [ ] **Recruiter:** Begin sourcing alternative opportunities if engagement ending
- [ ] **Operations:** Update contract status in system; calendar next milestone

## Key Success Indicators

**On-Time Metrics:**
- 100% of pre-start checklist items completed by Day -1
- Zero delays to Day 1 due to onboarding failures
- Same-day escalation response (within 30 min)

**Engagement Metrics:**
- 90-day retention rate > 90%
- 30-day performance rating (Green) > 85% of placements
- Time-to-productivity (first billable deliverable) < 5 days

**Satisfaction Metrics:**
- Client satisfaction rating (monthly) > 8/10
- Consultant satisfaction survey (30-day) > 7/10
- Timesheet compliance > 95% (on-time, accurate submission)
